# LLMNightRun 아키텍처

## 개요

LLMNightRun은 멀티 LLM 통합 자동화 플랫폼으로, 다양한 LLM(Large Language Model) 서비스를 효율적으로 통합하고 관리하기 위한 시스템입니다. 이 문서는 LLMNightRun의 시스템 아키텍처와 주요 구성 요소를 설명합니다.

## 시스템 아키텍처

LLMNightRun은 다음과 같은 주요 계층으로 구성됩니다:

1. **프론트엔드 레이어**: 사용자 인터페이스 제공
2. **백엔드 레이어**: API 및 비즈니스 로직 구현
3. **LLM 통합 레이어**: 다양한 LLM 서비스 연동
4. **데이터 저장소**: 질문, 응답, 코드 등의 데이터 저장
5. **문서 자동화 시스템**: 코드 분석 및 문서 생성

### 아키텍처 다이어그램

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   프론트엔드     │     │     백엔드       │     │  외부 LLM 서비스 │
│  (Next.js/React) │◄───►│   (FastAPI)     │◄───►│ (OpenAI, Claude) │
└─────────────────┘     └────────┬────────┘     └─────────────────┘
                                 │                        ▲
                                 ▼                        │
                        ┌────────────────┐       ┌────────────────┐
                        │  데이터베이스   │       │   로컬 LLM     │
                        │   (SQLite)     │       │   서비스       │
                        └────────────────┘       └────────────────┘
```

## 주요 컴포넌트

### 프론트엔드

- **기술 스택**: Next.js, React, TypeScript, Material-UI
- **주요 기능**:
  - 사용자 인터페이스 제공
  - LLM과의 대화 인터페이스
  - 코드 생성 및 관리 화면
  - 문서 자동화 관리 인터페이스
  - GitHub 연동 설정

### 백엔드

- **기술 스택**: FastAPI, Python
- **주요 모듈**:
  - `api/`: RESTful API 엔드포인트
  - `services/`: 비즈니스 로직 모듈
  - `models/`: 데이터 모델 정의
  - `database/`: 데이터베이스 연결 및 관리
  - `tool/`: 유틸리티 함수 및 도구
  - `agent/`: 자동화 에이전트 구현
  - `llm.py`: LLM 서비스 통합 모듈

### LLM 통합 레이어

- **지원 모델**:
  - OpenAI GPT 모델 (GPT-3.5, GPT-4)
  - Anthropic Claude 모델
  - 로컬 LLM 모델
- **통합 방식**:
  - RESTful API 호출
  - 스트리밍 응답 처리
  - 컨텍스트 관리
  - 프롬프트 최적화

### 데이터 저장소

- **기본 데이터베이스**: SQLite
- **주요 테이블**:
  - Questions: 사용자 질문 저장
  - Responses: LLM 응답 저장
  - CodeEntries: 생성된 코드 관리
  - Agents: 에이전트 정보 관리
  - Documents: 생성된 문서 관리

### 문서 자동화 시스템

- **기술 스택**: Python
- **주요 모듈**:
  - `code_analyzer.py`: 코드 분석 기능
  - `document_builder.py`: 문서 생성 기능
  - `git_handler.py`: GitHub 연동 관리
  - `llm_client.py`: LLM을 활용한 문서 생성
  - `workflow.py`: 문서화 워크플로우 관리

## 주요 워크플로우

### 1. 질문-응답 워크플로우

1. 사용자가 프론트엔드에서 질문 입력
2. 프론트엔드가 백엔드 API 호출
3. 백엔드에서 적절한 LLM 서비스 선택
4. LLM에 질문 전송 및 응답 수신
5. 응답을 데이터베이스에 저장
6. 사용자에게 결과 표시

### 2. 코드 생성 워크플로우

1. 사용자가 코드 생성 요청
2. 코드 생성에 특화된 프롬프트 구성
3. 선택된 LLM에 프롬프트 전송
4. 생성된 코드 검증 및 정제
5. 코드를 데이터베이스에 저장
6. 사용자에게 결과 표시

### 3. 문서 자동화 워크플로우

1. 사용자가 문서 생성 요청
2. 코드 분석기가 프로젝트 코드 분석
3. 분석 결과를 기반으로 문서 템플릿 선택
4. LLM을 사용하여 템플릿에 내용 채우기
5. 생성된 문서 저장
6. GitHub에 문서 커밋 및 푸시 (설정된 경우)

## 설계 원칙

LLMNightRun의 아키텍처는 다음과 같은 설계 원칙을 따릅니다:

1. **모듈성**: 각 컴포넌트는 독립적으로 개발 및 테스트 가능
2. **확장성**: 새로운 LLM 서비스 및 기능을 쉽게 추가 가능
3. **유연성**: 다양한 사용 사례와 요구사항에 맞게 구성 가능
4. **보안성**: 민감한 API 키 및 토큰의 안전한 관리
5. **효율성**: 리소스를 효율적으로 사용하여 최적의 성능 제공

## 향후 확장 계획

- **클러스터링 지원**: 대규모 배포를 위한 분산 아키텍처
