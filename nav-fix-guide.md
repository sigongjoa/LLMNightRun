# 네비게이션 메뉴 수정 가이드

이 문서는 LLMNightRun 애플리케이션의 상단 네비게이션 메뉴가 표시되지 않는 문제를 해결하기 위한 변경 사항을 설명합니다.

## 문제 원인

- `_app.tsx` 파일에 `Layout` 컴포넌트가 적용되지 않아 네비게이션 메뉴가 표시되지 않았습니다.
- 애플리케이션 전체에 공통 레이아웃이 적용되지 않아 각 페이지에서 네비게이션 메뉴가 누락되었습니다.

## 주요 변경 사항

1. **레이아웃 적용**
   - `_app.tsx` 파일에 `Layout` 컴포넌트를 추가하여 모든 페이지에 적용
   - 페이지별로 커스텀 레이아웃을 지정할 수 있는 옵션 추가

2. **스타일 개선**
   - 네비게이션 메뉴의 스크롤 동작 개선
   - 모바일 환경에서 햄버거 메뉴 버튼 스타일 조정
   - 스크롤바 숨김 처리로 깔끔한 UI 제공

3. **중복 요소 방지**
   - 여러 번 렌더링될 때 중복된 헤더 요소가 생성되는 문제 해결
   - 클래스 기반 숨김 처리로 DOM 구조 보존하며 시각적 오류 방지

4. **반응형 디자인 강화**
   - 작은 화면에서 메뉴 버튼이 잘 표시되도록 개선
   - 가로 스크롤 지원으로 모든 메뉴 항목에 접근 가능

## 구체적인 코드 변경

### 1. Layout 컴포넌트 적용
- `_app.tsx`에 Layout 컴포넌트를 가져와서 모든 페이지에 적용
- 페이지별 커스텀 레이아웃 지원을 위한 `getLayout` 패턴 구현

### 2. 스타일 업데이트
- 네비게이션 메뉴의 스크롤 동작 개선
- 모바일 환경에서의 UI 개선
- 글로벌 CSS 파일에 네비게이션 관련 스타일 추가

### 3. 중복 요소 처리 로직 개선
- 중복된、헤더 요소를 감지하고 클래스 기반으로 숨김 처리
- 네비게이션 버튼 활성화 상태 관리 개선

## 적용 방법

애플리케이션을 재시작하여 변경 사항을 적용하세요:

```bash
# 프론트엔드 개발 서버 시작
npm run dev

# 백엔드 서버 시작
uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

## 추가 개선 가능 사항

1. **페이지별 타이틀 설정**: 현재 페이지를 더 잘 나타내기 위한 동적 페이지 타이틀 구현
2. **활성 메뉴 강조**: 현재 페이지에 해당하는 메뉴 항목 강조 표시 개선
3. **메뉴 권한 관리**: 사용자 권한에 따른 메뉴 표시/숨김 기능 추가
