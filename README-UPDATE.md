# LLMNightRun 백엔드 연결 문제 해결 업데이트

이 업데이트는 프론트엔드와 백엔드 간의 연결 문제를 해결하고, 백업 서버 기능을 추가하여 API 서버 가용성을 향상시킵니다.

## 주요 변경사항

### 1. 백엔드 서버 개선
- 건강 상태 확인 엔드포인트 (`/health-check`) 추가
- 오류 처리 개선
- GitHub 저장소 API 수정 - 사용자 객체 대신 ID와 이름 사용

### 2. 프론트엔드 API 클라이언트 개선
- 자동 서버 장애 조치(failover) 구현
- 기본 서버(8000) 및 백업 서버(8001) 지원
- 주기적인 서버 상태 확인 및 자동 전환
- 네트워크 오류 처리 개선
- 데이터 로드 실패에 대한 폴백 메커니즘

### 3. 시스템 상태 모니터링
- 새로운 `SystemStatusPanel` 컴포넌트 추가
- API 및 서비스 연결 상태 실시간 표시
- 서버 상태 수동 새로고침 기능

### 4. 이중 서버 구성
- 동시에 두 개의 서버 인스턴스 실행 스크립트 추가 (`start-server.bat`, `start-server.sh`)
- 한 서버에 문제 발생 시 자동으로 다른 서버로 전환

## 설치 및 실행 방법

### 백엔드 서버 실행
Windows:
```
cd D:\LLMNightRun\backend
start-server.bat
```

Linux/Mac:
```
cd /path/to/LLMNightRun/backend
chmod +x start-server.sh
./start-server.sh
```

### 프론트엔드 실행
```
cd D:\LLMNightRun\frontend
npm run dev
```

## 주요 파일 변경 내역

1. **백엔드**:
   - `main.py`: 헬스체크 엔드포인트 추가 및 오류 처리 개선
   - `start-server.bat`/`start-server.sh`: 이중 서버 실행 스크립트

2. **프론트엔드**:
   - `utils/api.ts`: 서버 장애 조치 및 오류 처리 개선
   - `components/SystemStatusPanel.tsx`: 새로운 시스템 상태 모니터링 컴포넌트
   - `pages/index.tsx`: 대시보드 업데이트 및 상태 패널 통합
   - `.env.local`: 백업 서버 URL 추가

## 문제 해결 테스트 방법

1. 프론트엔드와 백엔드를 모두 실행합니다.
2. 대시보드 페이지에서 시스템 상태 패널이 "연결됨"을 표시하는지 확인합니다.
3. 기본 서버를 중지하여 장애 시나리오를 테스트합니다:
   - Windows: 포트 8000의 CMD 창을 닫습니다.
   - Linux/Mac: `kill $(lsof -ti:8000)` 명령을 실행합니다.
4. 프론트엔드는 자동으로 백업 서버(8001)로 전환되어야 합니다.
5. 시스템 상태 패널이 "백업 서버"로 업데이트되어야 합니다.

## 제한 사항

- 두 서버가 독립적인 데이터베이스를 사용하므로, 한 서버에서 다른 서버로 전환 시 일부 데이터가 동기화되지 않을 수 있습니다.
- 양쪽 서버 모두 사용할 수 없는 경우 적절한 오류 메시지가 표시되지만, 오프라인 기능은 제공되지 않습니다.

## 향후 개선 사항

- 데이터베이스 복제 또는 공유 스토리지를 통한 서버 간 데이터 동기화
- 서버 상태 모니터링 대시보드 개선
- 오프라인 모드 및 자동 재연결 기능 추가
- 추가 백업 서버 구성 옵션
